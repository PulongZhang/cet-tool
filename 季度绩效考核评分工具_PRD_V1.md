## 1. 项目背景与目标

### 1.1 背景现状

软件研发中心每季度进行绩效考核，当前流程依赖线下 Excel 表格流转，主要痛点如下：

* 各部门各层级表格传递不及时，可能会出现表格传递错误或者沿用陈旧表格信息的情况；
* 团队主管下发表格需拆分，回收各小组打分需重新合并（复制粘贴汇总核算比例、数据错位风险高、反复调整耗时久），耗费团队主管的精力
* 各个环节都用excel核算，HR 需手工合并后计算加权分和排名
* 等级定级规则复杂（分组排序 + 百分比映射），人工执行易出错且难以追溯
* 间接上级/部门负责人难以整体了解整个团队各等级分布，缺乏系统化的比例校验工具

### 1.2 项目目标

本阶段聚焦核心目标：以一套线上评分工具替代现有手工 Excel 流转，解放评估人员的重复性表格操作时间。

## 3. 核心业务流程

### 3.1 整体流程图

|  |  |
| --- | --- |
| **阶段** | **操作步骤与责任人** |
| **阶段一 准备工作** | ① 管理员创建考核周期（如"2026-Q2"），设定起止日期 → 状态：准备中  ② 管理员通过 Excel 模板导入参与本次考核的人员名单（工号、姓名、序列、职级、部门、各级评分人ID）  ③ 系统根据人员数据自动生成每位员工的评分记录，状态初始化为"待自评" |
| **阶段二 员工自评** | ①员工登录系统 → 查看待自评任务 → 填写季度工作总结（文本）  ② 对3个主观维度逐一选择等级（A+/A/B+/B/B-/C/D），系统展示对应评价标准供参考  ③ 保存草稿（可多次修改）或 提交（锁定自评内容，状态变为"待直接上级评分"）  **⚠️ 已提交不可自行修改，需联系直接上级或HR撤回** |
| **阶段三**  **直接上级评分** | ① 直接上级登录 → 查看"待评分"下属列表（显示下属姓名、是否已自评）  ② 点击进入某位下属的评分页 → 左侧展示该员工的自评内容和自评分数（只读）  ③ 右侧对3个主观维度逐一打分 + 初始总分评分+可选填写评语（A及CD必填）  ④ 保存草稿锁定评分，所有下属评分完毕进行提交，状态变为"待间接上级审阅"  **⚠️ 已提交不可自行修改，需联系间接上级或HR撤回** |
| **阶段四 间接上级评分** | ①间接上级登录 → 查看"待我审阅"团队列表，看到所有一级评分人已完成评分的下属汇总  ②进入"初始评级比例校验"面板，查看当前各等级人数分布（A+/A/B+/B/B-/C/D 及各占比%），系统对照参考比例高亮显示明显偏差的等级  ③间接上级查看各一级评分人的评分情况，核实是否存在普遍偏高/偏低的情况，并综合判断是否需要调整  ④如需调整：在人员列表中选择具体人员 → 修改其建议等级 → 必填"调整原因"（系统记录：调整人工号+姓名、调整前等级、调整后等级、调整原因、调整时间）  ⑤间接上级确认完成审阅后，点击"提交审阅结果" → 当前管辖范围的评分记录状态变更为"待部门负责人审阅"  **⚠️若间接上级本身也是某些员工的直接上级，则其作为直接上级填写的评分在阶段三已提交，此处以间接上级身份进行整体校验；两个角色操作互不干扰**  **⚠️ 调整记录不可删除，仅可追加新的调整记录（保留完整调整历史）**  **⚠️ 已提交不可自行修改，需联系部门负责人或HR撤回** |
| **阶段五 部门负责人最终评分** | ①部门负责人登录 → 查看"待我最终审阅"的本部门全员评分总览  ②查看"本部门初始评级比例校验"面板：展示非管理序列所有员工各等级人数及占比，系统对照参考比例高亮超出参考区间的等级  ③部门负责人查看具体人员的评分详情（含直接上级评分和间接上级调整记录，只读），综合判断是否需要进一步调整  ④如需调整：选择人员 → 修改最终等级 → 必填"调整原因"（系统记录：部门负责人工号+姓名、调整前等级、调整后等级、调整原因、调整时间）  ⑤部门负责人确认完成后，点击"确认提交本部门评分" → 本部门全员记录状态变更为"待HR计算"  **⚠️部门负责人的调整是最终调整，HR在阶段六执行加权计算前，以部门负责人确认后的等级作为主观维度最终评分**  **⚠️所有调整操作均留痕，导出结果时包含完整的调整历史**  **⚠️已提交不可自行修改，需联系HR撤回** |
| **阶段六 数据处理** | ① HRBP 下载客观数据导入模板 → 填写勤奋/纪律/学习数据 → 上传  ② 系统校验数据格式、匹配员工 → 显示校验结果（成功N条/失败N条及原因）→ 支持补录修正后再次导入  ③ 系统按预设规则自动将原始数值转换为勤奋/纪律/学习三个客观维度的等级  ④ HRBP 审查客观等级转换结果 → 可对异常值做人工修正（必填修正原因）  ⑤ 点击"执行加权计算" → 系统自动完成：主观分采用部门负责人最终确认后的等级× 权重 + 客观分等级×权重 → 加权总分 → 分组排序 → 百分比映射定级  ⑥ 计算完成后状态变为"已完成"，HRBP 可查看所有人的详细计算明细  ⑦生成全体员工（除去管理序列）评分结果占比比例数据 |
| **阶段七 结果输出** | ①HRBP 查看计算结果总览表（含排名、等级比例分布统计）  ②若整体等级比例分布不符合要求，由HRBP与各部门经理做沟通提出调整建议  ③HRBP将各部门评分初评结果下载导出返回给各部门经理，由部门经理对总分评级进行微调后返给HRBP→HRBP在系统中做最终微调标注，生成最终版结果  ④一键导出 Excel 结果文件（含所有字段：人员基本信息、自评分数、上级分数、客观数据、加权分、排名、等级）  ⑤ 导出的 Excel 作为正式考核结果依据，后续通过线下流程通知部门 |

### 3.2 工作流状态机（关键设计决策）

| --- | --- | --- | --- |
| **当前状态** | **可执行的操作** | **操作后的下一状态** | **备注** |
| 待自评 | 员工：填写自评 → 保存草稿 / 提交 | 草稿 / 待上级评分 | 草稿状态下员工可随时修改 |
| 草稿 | 员工：继续编辑保存 / 提交 | 草稿 / 待上级评分 | - |

| 评分中（直接上级） | 直接上级：填写评分 → 保存草稿 / 提交 | 评分草稿 / 待间接上级审阅 | 上级可查看员工自评内容作参考 |

| 待部门负责人审阅 | 部门负责人：查看全员评分 → 校验比例 → 调整或确认提交 | 待HR计算 | 最终调整，全程留痕 |
| 评分草稿 | 上级：继续编辑 / 提交 / 撤回他人已提交评分 | 评分草稿 / 待计算 / 待上级评分 | 撤回需填写原因；仅上级本人或HRBP有撤回权限 |
| 待HR计算 | HRBP：导入客观数据 / 执行计算 | 待计算 / 已完成 | 此阶段上级评分已锁定 |
| 已完成 | HRBP：查看明细 / 导出Excel / 线下最终微调 / 手动调级/导出最终结果 | 已完成 | 最终状态；所有操作记入审计日志 |
| *💡 核心原则：(1) 提交即锁定；(2) 撤回操作强制记录原因和时间戳；(3) HRBP 拥有最高操作权限，可撤回任何环节的数据。* | | | |

### 3.3 比例校验规则说明

间接上级（阶段四）和部门负责人（阶段五）在校验比例时，系统展示以下信息：

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **校验项** | | **计算方式** | **参考区间** | **展示方式** |
| **A+占比** | | 当前A+人数 ÷ 分组总人数 | 参考值 ≈ 5%（允许 ±2%） | 实际值与参考值对比，偏差>±2%则橙色高亮  超出则红色高亮，提示审慎处理 |
| **A占比** | | 当前A人数 ÷ 分组总人数 | 参考值 ≈ 15%（允许 ±3%） |
| **B+占比** | | 当前B+人数 ÷ 分组总人数 | 参考值 ≈ 30%（允许 ±5%） |
| **B占比** | | 当前B人数 ÷ 分组总人数 | 参考值 ≈ 35%（允许 ±5%） |
| **B-占比** | | 当前B-人数 ÷ 分组总人数 | 参考值 ≈ 10%（允许 ±3%） |
| **C+D占比** | | 当前C+D人数 ÷ 分组总人数 | 参考值 ≈ 5%（允许 ±2%） |
| *⚠️ 重要说明：各等级人数统计均不含管理序列人员；比例校验为"参考性校验"而非"强制拦截"。系统会给出高亮提示，但不会阻止间接上级/部门负责人直接提交。最终决策权在负责人手中。待确认事项9.1中需明确容差参考值是否采用上述建议值。* | | | |

## 4. 核心数据模型与评分规则

### 4.1 人员分类体系

本系统中参与考核的人员分为以下三类，分别独立排序定级：

|  |  |  |
| --- | --- | --- |
| **分组** | **包含人员** | **排序方式** |
| 管理序列 | 具有直接下属管理职责的管理岗位人员 | 单独一组内排序 |
| 员工序列-P1-3职级 | 非管理岗的初级人员（原D/E职级） | 单独一组内排序 |
| 员工序列-P4-10职级 | 非管理岗的中高级人员（原B/C职级） | 单独一组内排序 |

### 4.2 评分维度体系

【重要规则说明】关于主观维度的最终评分，规则如下：

• 加权计算时，主观维度的得分 100% 采用部门负责人最终确认后的等级

• 直接上级原始评分、间接上级调整后等级、部门负责人最终等级，全程记录留痕

• 自评分数作为参考信息供各级评分时查阅，但不参与最终加权计算：

|  |  |  |  |
| --- | --- | --- | --- |
| **维度类型** | **管理序列** | **员工序列-P4-10** | **员工序列-P1-3** |
| 主观维度1 | 工作能力和方法（20%） | 产出和质量（30%） | 产出和质量（30%） |
| 主观维度2 | 团队业绩和产出（30%） | 主动承担（25%） | 主动承担（20%） |
| 主观维度3 | 个人关键任务（30%） | 易用性和可维护（30%） | 易用性和可维护（30%） |
| 客观-勤奋 | 10% | 5% | 10% |
| 客观-纪律 | 5% | 5% | 5% |
| 客观-学习 | 5% | 5% | 5% |

### 4.3 分数映射（等级 → 数值）

|  |  |  |
| --- | --- | --- |
| **等级** | **分值** | **语义** |
| A+ | 100 | 工作表现卓越，在各方面均超越所在岗位组织要求，在核心领域做出突破性贡献，对战略目标达成或关键任务完成具有决定性影响，展现出卓越的价值导向与标杆行为 |
| A | 93 | 工作表现优秀，达到并经常性超出所在岗位组织要求，经常性主动承担并出色完成复杂挑战性任务，在关键工作中表现突出，显著超出常规期望 |
| B+ | 86 | 工作表现稳定可靠，能全面达成岗位职责与工作任务，工作质量与效率稳定达到或略高预期 |
| B | 80 | 工作表现良好，能达成岗位职责与工作任务要求，整体表现符合预期 |
| B- | 70 | 工作表现尚可，能基本完成本职工作，达到岗位大部分要求，在工作质量、效率或能力上有持续提升空间 |
| C | 60 | 工作表现未达到岗位要求，须制定改进计划 |
| D | 50 | 工作表现明显低于岗位要求 |

### 4.4 客观维度等级转换规则

客观数据（勤奋/纪律/学习）的原始数值需先转换为7级等级，再参与加权计算：

**【勤奋】基于该季度月均勤奋点数：**

|  |  |
| --- | --- |
| **季度勤奋点数合计** | **转换等级** |
| 月均勤奋点数＜11 | D |
| 11≤月均勤奋点数＜40 | C |
| 40≤月均勤奋点数＜60 | B |
| 月均勤奋点数≥ 60 | A |

**【纪律】基于季度累计异常次数（考勤异常+日志违规）：**

|  |  |
| --- | --- |
| **季度异常次数合计** | **转换等级** |
| 0 – 3次 | A+ |
| 4 – 6次 | A |
| 7 – 9次 | B |
| 10 – 12次 | C |
| ≥ 13次 | D |

**【学习】基于同一分组内的培训时长排名百分比：**

|  |  |  |
| --- | --- | --- |
| **培训时长排名百分比** | **转换等级** | **占比** |
| ≤ 5% | A+ | 5% |
| >5% 且 ≤ 20% | A | 15% |
| >20% 且 ≤ 50% | B+ | 30% |
| >50% 且 ≤ 85% | B | 35% |
| >85% 且 ≤ 95% | B- | 10% |
| >95% 且 ≤ 98% | C | 3% |
| > 98% | D | 2% |

### 4.5 排序与定级规则

各分组序列所有员工加权得分排序与评价定级对应关系如下：

|  |  |  |
| --- | --- | --- |
| **组内排名百分比** | **转换等级** | **理论占比** |
| ≤ 5% | A+ | 5% |
| >5% 且 ≤ 20% | A | 15% |
| >20% 且 ≤ 50% | B+ | 30% |
| >50% 且 ≤ 85% | B | 35% |
| >85% 且 ≤ 95% | B- | 10% |
| >95% 且 ≤ 98% | C | 3% |
| > 98% | D | 2% |

**【管理序列处理】**

管理序列人员在"管理序列"组内独立排序定级，不参与员工序列的总体/部门分布校验（V1 不含分布校验功能）。管理序列的最终等级由 HRBP 在结果审查时确认。

### 4.6 加权计算公式示例

以管理序列员工为例，完整计算过程如下：

最终得分 = 工作能力和方法(上级评分)×20% + 团队业绩和产出(上级评分)×30%

+ 个人关键任务(上级评分)×30% + 勤奋(客观等级分)×10%

+ 纪律(客观等级分)×5% + 学习(客观等级分)×5%

|  |
| --- |
| *💡 计算过程中系统展示完整明细：每项维度的等级、对应的数值、权重、加权贡献值，以及最终的加权和。HRBP 可逐人核对。* |
